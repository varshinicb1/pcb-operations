## State Management

**Riverpod** — all providers in same file as repository:
- `todayAttendanceProvider` → `StreamProvider<Attendance?>` — live attendance status
- `attendanceHistoryProvider` → `FutureProvider<List<Attendance>>` — 30-day history
- `branchAttendanceProvider` → `FutureProvider.family` — filtered by branch+date
- `currentEmployeeProvider` → `StreamProvider<Employee?>` — reactive profile
- `currentEmployeeRoleProvider` → `Provider<String>` — role for UI gating
- `goRouterProvider` → `Provider<GoRouter>` — auth-aware routing
- `leaveRepositoryProvider` → `Provider<LeaveRepository>`
- `employeeLeavesProvider` → `StreamProvider.family`

## Navigation

GoRouter with ShellRoute. Route table:

| Path | Screen | Auth |
|------|--------|------|
| `/splash` | Animated welcome | None |
| `/login` | Sign in + self-registration | None |
| `/home` | Admin Dashboard | Required |
| `/production` | Placeholder | Required |
| `/attendance` | Attendance screen | Required |
| `/employees` | Employee directory | Required |
| `/reports` | Attendance reports | Required |

Auth redirect: checks `authStateProvider` → if null redirects to `/login`, if authenticated redirects `/login` → `/home`.

## Data Flow

### Check-In Flow
```
User taps Check In
├── Step 1: GPS capture (Geolocator.high accuracy)
├── Step 2: Selfie (image_picker, CameraSource, quality:70, maxWidth:800)
├── Step 3: Confirm
│   ├── dart:convert → base64Encode(file bytes)
│   └── Firestore attendance/{uid}_{date} document
│       ├── employeeId, employeeName, branchId, date
│       ├── checkInTime, checkInLatitude, checkInLongitude
│       ├── selfieUrl (base64 string, ~80KB JPEG)
│       └── status: "present" | "late"
└── Admin dashboard streams updates in real-time
```

### Selfie Storage
- **No Firebase Storage** (Spark free tier limitation)
- Selfies stored as base64 in Firestore `attendance.selfieUrl` field
- Displayed via `SelfieImage` widget (base64Decode → Image.memory)
- 1MB Firestore document limit — photos compressed to ~80KB

## Firebase Setup

| Service | Status | Notes |
|---------|--------|-------|
| **Auth** | Enabled | Email/Password provider |
| **Firestore** | Mumbai (asia-south1) | Rules + 9 composite indexes deployed |
| **Storage** | Not used | Selfies via base64 in Firestore |
| **Functions** | Not used | Spark free tier |
| **Hosting** | Not configured | Future: web deployment |

### Security Rules (`firestore.rules`)
- Role-based access via `get()` on employee document (not custom claims)
- Attendance: create requires `employeeId == auth.uid`, update validates field immutability
- Employees: only admin can create/delete, self-update blocked on role field
- Default deny-all catch rule at bottom

### Firestore Collections
| Collection | Purpose | Key queries |
|-----------|---------|-------------|
| `employees` | Worker profiles | branchId+department+isActive |
| `attendance` | Daily check-in records | employeeId+date, branchId+date |
| `branches` | Branch locations | id |
| `buses` | Production tracking | branchId+status+deliveryDate |
| `leaves` | Leave requests | employeeId, branchId+status |

## Key Design Decisions

1. **No Google Fonts** — system fonts for instant load, no network dependency
2. **No Firebase Storage** — selfies as base64 in Firestore (free tier)
3. **No Cloud Functions** — role checks via Firestore document `get()` in rules
4. **No freezed** — plain classes with `json_serializable` for code gen, `copyWith` by hand
5. **No riverpod_generator** — hand-written providers for build tool compatibility
6. **GoRouter** — declarative routing with auth redirect, no navigator.push for main flows
7. **compileSdk 36** — required by modern AndroidX libraries
8. **minSdk 21** — supports Android 5.0+
